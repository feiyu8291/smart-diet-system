## 什么是 Awesome DESIGN.md？

将一份 `DESIGN.md` 复制到您的项目根目录下，然后告诉您的 AI 编程代理（例如 Antigravity）**“帮我构建一个看起来像这样的页面”**，AI 就能以此生成出与该品牌视觉设计语言（设计系统、令牌和限制规则）高度一致的高质量
UI。

它不仅捕获表面样式，而且还捕获了真正的设计深度——包括经过分析的视觉模式、颜色/圆角/间距令牌（Tokens）以及排版规则——用于高质量的 AI UI 生成。

## 什么是 DESIGN.md？

[DESIGN.md](https://stitch.withgoogle.com/docs/design-md/overview/) 是由 Google Stitch 提出的一项新概念。它是一个**纯文本设计系统规范文档**，AI 代理可以读取它来生成风格一致的 UI。

它仅仅是一个普通的 Markdown 文件。不需要 Figma 导出，不需要 JSON schemas，不需要特殊的工具链。只需将其丢入项目根目录中，任何 AI 编码代理或 Google Stitch 都能立即理解您的 UI 应该如何呈现。由于 Markdown
是大语言模型（LLM）最擅长读取的排版格式，因此无需任何解析或配置。

| 文件          | 谁来阅读它   | 它定义了什么           |
|-------------|---------|------------------|
| `AGENTS.md` | 编码代理    | 如何构建、运行和维护项目逻辑   |
| `DESIGN.md` | 设计/前端代理 | 项目的外观、色彩、排版与视觉质感 |

**本项目提供了一系列提取自真实主流网站的开箱即用的 DESIGN.md 文件。**

## 每一份 DESIGN.md 中都包含什么

本仓库中收集的每个文件均严格遵循 [Google Stitch DESIGN.md 规范格式](https://stitch.withgoogle.com/docs/design-md/specification/) 编写：

| 编号 | 章节名称                                | 它捕获的核心设计内涵                    |
|----|-------------------------------------|-------------------------------|
| 1  | 视觉主题与氛围 (Visual Theme & Atmosphere) | 视觉情感基调、信息密度、设计理念哲学            |
| 2  | 调色板与配色角色 (Color Palette & Roles)    | 包含十六进制 Hex 的语义命名与对应的业务功能色     |
| 3  | 排版规则 (Typography Rules)             | 所用字族（Font families）以及全层级排版对照表 |
| 4  | 组件规范样式 (Component Stylings)         | 包含各交互状态的按钮、卡片、输入框、导航栏规范       |
| 5  | 布局与结构原则 (Layout Principles)         | 间距缩放级、网格规范、对白边/留白的设计哲学        |
| 6  | 深度与高度等级 (Depth & Elevation)         | 阴影层级系统，各物理卡片所浮现的海拔高度          |
| 7  | 设计守则与禁止红线 (Do's and Don'ts)         | 品牌设计的避坑指南与错误设计防范反例            |
| 8  | 响应式媒体查询行为 (Responsive Behavior)     | 适配断点、最小触摸交互热区、不同视口折叠策略        |
| 9  | 代理专属 Prompt 引导 (Agent Prompt Guide) | 针对大模型的颜色速查与现成的系统初始化提示词        |

每家企业的规范均包含以下关联资产（存放于其各自的原子文件夹内）：

- `DESIGN.md`: 设计系统说明书文件（直接给 AI 开发助手的核心）。
- `preview.html`: 网页视觉展示效果（包含了调色板、字号、基础按钮卡片等）。
- `preview-dark.html`: 同样具有完整组件展示的暗黑模式预览网页。

## 如何使用它

1. 选择一个适合您定位的公司，将对应的 `*.md` 设计文档复制到您开发项目的根目录下，重命名为 `DESIGN.md`。
2. 在开发该页面时，直接提示您的 AI 编码助手遵循此规则（“请读取根目录下的 `DESIGN.md` 规范，帮我构建一个看起来完全符合其色彩和排版规范的 UI 页面”）。
